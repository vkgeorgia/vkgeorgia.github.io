# Спецификация: Цифровой Аватар для Консалтинговых Услуг (Low-Code подход)

## Обзор проекта

Цифровой аватар - это AI-система, которая представляет пользователя в общении с рекрутерами и потенциальными заказчиками консалтинговых услуг. Аватар использует информацию об опыте работы, проектах и навыках пользователя для ведения профессиональных диалогов.

**Подход:** Использование готовых low-code решений для быстрого создания MVP с минимальными трудозатратами и максимальной эффективностью.

## Цели проекта

1. **Автоматизация первичного общения** - аватар может отвечать на базовые вопросы о квалификации и опыте
2. **Презентация экспертизы** - эффективно представлять навыки и достижения пользователя
3. **Фильтрация запросов** - определять релевантность предложений и проектов
4. **Масштабирование присутствия** - участвовать в множественных диалогах одновременно

## Функциональные требования

### 1. Управление профилем
- **Хранение данных о пользователе:**
  - Профессиональный опыт (должности, компании, периоды работы)
  - Образование и сертификации
  - Навыки и компетенции
  - Проекты и достижения
  - Портфолио работ
  - Контактная информация

### 2. Система общения
- **Поддержка каналов связи:**
  - Текстовые сообщения (Telegram, WhatsApp, Email - будущее)
  - Веб-чат (текущая реализация)

### 3. Интеллектуальные возможности
- **Понимание контекста:**
  - Анализ запросов рекрутеров и заказчиков
  - Определение типа взаимодействия (поиск работы vs. консалтинг)
  - Оценка релевантности предложений

- **Генерация ответов:**
  - Персонализированные ответы на основе профиля
  - Адаптация стиля общения под собеседника
  - Предоставление релевантных примеров работ

### 4. Бизнес-логика
- **Оценка предложений:**
  - Анализ соответствия требованиям пользователя
  - Оценка бюджета и временных рамок
  - Определение приоритетности запросов

- **Сбор информации:**
  - Вопросы о деталях проекта
  - Уточнение требований
  - Сбор контактных данных

## Технические требования

### 1. Архитектура
- **LLM:** Google Gemini (`gemini-2.5-flash`)
- **RAG:** Custom Python implementation with local/Google Drive knowledge base
- **Планирование:** Google Calendar Appointment Schedules
- **База знаний:** Google Drive + локальные файлы (fallback)
- **Backend:** Python FastAPI с WebSocket
- **Frontend:** HTML/CSS/JavaScript (чистый текстовый чат)
- **Hosting:** GitHub Pages (frontend), Google Cloud Run (backend - будущее)

### 2. Интеграции
- **Мессенджеры:** Telegram Bot API, WhatsApp Business API
- **Email:** SMTP/IMAP для работы с почтой
- **CRM:** Интеграция с существующими CRM-системами
- **Календарь:** Google Calendar, Outlook для планирования встреч

### 3. Безопасность
- **Аутентификация:** OAuth 2.0, JWT токены
- **Шифрование:** End-to-end шифрование сообщений
- **Конфиденциальность:** Локальное хранение чувствительных данных

## Архитектура системы

### Компоненты
```
Пользователь → Web-интерфейс → FastAPI Backend → RAG Service → Gemini API
                                       ↓
                                 Google Drive API (Knowledge Base)
```

### Рабочий процесс
1. **Пользователь** отправляет запрос через веб-чат
2. **FastAPI Backend** получает сообщение через WebSocket
3. **RAG Service** загружает контекст из Google Drive или локальных файлов
4. **Gemini API** генерирует ответ на основе контекста
5. **Backend** отправляет ответ обратно пользователю через WebSocket

### Преимущества подхода
- **Быстрый MVP:** Базовая функциональность готова за 1-2 недели
- **Гибкость:** Легко добавлять новые источники знаний
- **Надёжность:** Offline fallback при недоступности API
- **Масштабируемость:** Готов к деплою на Google Cloud Run

## Пользовательские сценарии

### Сценарий 1: Общение с рекрутером
1. Рекрутер отправляет сообщение с вакансией
2. Аватар анализирует требования и соответствие профилю
3. Аватар задает уточняющие вопросы о компании и условиях
4. Аватар предоставляет релевантную информацию о кандидате
5. При заинтересованности - передает контактные данные

### Сценарий 2: Общение с потенциальным заказчиком
1. Заказчик описывает потребность в консалтинге
2. Аватар анализирует проект и соответствие экспертизе
3. Аватар задает вопросы о бюджете, сроках, деталях
4. Аватар презентует релевантный опыт и подходы
5. Аватар координирует дальнейшее общение

## Критерии успеха

1. **Качество общения:** Естественность и профессионализм диалогов
2. **Точность информации:** Корректное представление навыков и опыта
3. **Эффективность:** Сокращение времени на первичное общение на 70%
4. **Конверсия:** Увеличение количества релевантных предложений на 50%

## Ограничения и риски

1. **Этические аспекты:** Прозрачность использования AI в общении
2. **Качество данных:** Зависимость от полноты и актуальности профиля
3. **Контекстные ошибки:** Возможность неправильной интерпретации запросов
4. **Техническая надежность:** Обеспечение стабильной работы системы

## Метрики и аналитика

- Количество обработанных запросов
- Процент релевантных ответов
- Время отклика на сообщения
- Конверсия в реальные встречи/сделки
- Удовлетворенность собеседников
